# Claude Code CLI — 架构分析

> 分析对象：`cc/src/`（TypeScript 源码）
> 技术栈：Bun · TypeScript · React + Ink（TUI）· Anthropic SDK · Zod

---

## 一、整体结构

Claude Code CLI 的完整 TypeScript 源码，主要分为以下几个层次：

1. 核心循环：query.ts 是心脏——构建 API 请求 → 流式调用 Claude → 解析 tool_use → 执行工具 → 回收 tool_result → 循环直到 end_turn
2. 工具体系：30+ 工具统一实现 ToolDef 接口，每个工具是一个 AsyncGenerator（支持流式进度）。ToolUseContext 像一个依赖注入容器，贯穿整个调用链。
3. 多 Agent：AgentTool 是多智能体的入口——产生子 Agent、分配隔离上下文（messages + 文件缓存 + worktree），通过 task-notification XML 把结果注入主会话。Coordinator Mode 下主 Agent 专职编排、不直接干活。
4. 状态管理：纯函数 pub/sub Store + React useSyncExternalStore，AppState 存储消息历史、任务注册表、权限上下文、待办列表等所有可变状态。
5. 权限系统：多层规则（全局→项目→本地→MDM）+ Hook 脚本，支持 default / auto / bypass / plan 四种模式。
6. Skill 系统：内置（编译进二进制）、磁盘（~/.claude/skills/）、MCP 三种来源，可 inline 继续或 fork 子 Agent 执行。

```
cc/src/
├── entrypoints/        # 入口点（CLI / SDK / MCP）
├── query.ts            # 核心 Agent 循环
├── QueryEngine.ts      # 高阶会话协调器
├── Tool.ts             # Tool 抽象 + ToolUseContext
├── Task.ts             # Task 生命周期类型
├── tools.ts            # 工具池装配
├── commands.ts         # 斜杠命令注册
│
├── tools/              # 30+ 内置工具实现
├── tasks/              # 后台任务管理
├── commands/           # 60+ 斜杠命令
├── skills/             # Skill 系统
├── services/           # 基础服务（API / MCP / 分析 / 压缩 / 记忆）
├── bridge/             # 远程执行桥（CCR）
├── coordinator/        # 多 Agent 协调器模式
├── state/              # 全局状态（AppStateStore）
├── components/         # React/Ink UI 组件
├── ink/                # Ink TUI 框架适配
├── utils/              # 通用工具函数
└── context/            # React Context
```

## 二、核心抽象

### 2.1 Tool（工具）

文件：`Tool.ts`

```
ToolUseContext          # 工具运行时上下文（贯穿整个调用链）
├── options             # 配置：commands / tools / model / mcpClients
├── abortController     # 取消信号
├── readFileState       # 文件状态 LRU 缓存
├── getAppState()       # 只读全局状态
├── setAppState()       # 更新全局状态
├── permissionContext   # 权限规则集（ToolPermissionContext）
├── messages            # 当前会话消息列表（Message[]）
├── agentId?            # 子 Agent 标识（主线程为 undefined）
└── setToolJSX?         # 向 TUI 注入 React 节点

ToolPermissionContext
├── mode                # default | auto | bypass | plan
├── alwaysAllowRules    # 自动放行规则
├── alwaysDenyRules     # 自动拒绝规则
└── alwaysAskRules      # 强制询问规则
```

每个工具必须实现 `ToolDef`（`buildTool()` 构建）：
- `name / description / inputSchema`（Zod）
- `call(input, ctx)` → `AsyncGenerator<StreamEvent>`
- `getActivityDescription?` → 进度描述
- `renderToolUseMessage?` / `renderToolResultMessage?` → TUI 渲染

### 2.2 Task（任务）

文件：`Task.ts`、`tasks/types.ts`

```
TaskType
├── local_agent         # 进程内子 Agent（LocalAgentTask）
├── remote_agent        # 远程 CCR Agent（RemoteAgentTask）
├── local_bash          # 后台 Bash 命令（LocalShellTask）
├── in_process_teammate # Swarm 队友（InProcessTeammateTask）
├── local_workflow      # 本地工作流（LocalWorkflowTask）
├── monitor_mcp         # MCP 监控（MonitorMcpTask）
└── dream               # 自动 Dream（DreamTask）

TaskStatus: pending → running → completed | failed | killed

TaskStateBase
├── id / type / status / description
├── toolUseId           # 关联的 tool_use block ID
├── startTime / endTime
└── outputFile          # 磁盘输出路径（TaskOutput 工具读取）
```

### 2.3 Message（消息）

消息类型覆盖：`UserMessage`、`AssistantMessage`、`ProgressMessage`、`SystemMessage`、`AttachmentMessage`、`TombstoneMessage`、`ToolUseSummaryMessage`、`SystemCompactBoundaryMessage`

消息流动方向：用户输入 → API 请求 → 流式事件 → 存入 `AppState.messages`

## 三、数据流

### 3.1 主流程

```
用户输入（键盘 / stdin / SDK）
         │
         ▼
processUserInput()          [utils/processUserInput/]
  ├── 解析 / 验证输入
  ├── 处理附件（图片、文件）
  └── 检测斜杠命令

         │
         ▼
QueryEngine.processInput()  [QueryEngine.ts]
  ├── 加载记忆（CLAUDE.md、~/.claude/memory/）
  ├── 构建 SystemPrompt
  ├── 初始化 ToolUseContext
  └── 调用 query()

         │
         ▼
query()                     [query.ts]  ← 核心循环
  ├── 构造 API 请求（messages + tools + system）
  ├── queryModelWithStreaming()  [services/api/claude.ts]
  │     └── Anthropic SDK streaming
  ├── 解析流式事件（text / tool_use / stop_reason）
  ├── 权限检查 canUseTool()
  ├── 执行工具 tool.call()
  ├── 收集 tool_result
  └── 循环直到 stop_reason = "end_turn"

         │
         ▼
setAppState()               → React 重渲染（Ink TUI）
```

### 3.2 工具执行细节

```
Assistant 返回 tool_use block
         │
         ▼
findToolByName(name, tools)
         │
         ▼
canUseTool(tool, input, ctx)      [hooks/useCanUseTool.ts]
  ├── alwaysAllow 规则 → 自动通过
  ├── alwaysDeny 规则  → 自动拒绝
  ├── Hook 脚本（pre-tool-use）
  ├── 权限模式检查（auto / bypass）
  └── 显示 UI 对话框（交互模式）/ 自动拒绝（无头模式）
         │
         ▼
tool.call(input, ctx)
  └── AsyncGenerator<StreamEvent>
         │
         ▼
累积 tool_result → 追加到 messages → 下一轮 API 请求
```

### 3.3 上下文压缩

```
query() 循环内，每轮检测：
  autoCompactIfNeeded()
  ├── token 数超阈值 → 触发自动压缩
  ├── microcompact（小规模内联压缩）
  └── 完整 compact（调用 Claude 生成摘要）
       └── 插入 SystemCompactBoundaryMessage 做水位线
```

## 四、多 Agent 架构

### 4.1 普通模式 vs 协调器模式

```
普通模式（默认）
  └── 单一 Agent 循环，AgentTool 可产生后台子 Agent

协调器模式（CLAUDE_CODE_COORDINATOR_MODE=1）
  └── 主 Agent 专职编排，不直接执行任务
      └── 通过 AgentTool 生成 Worker
          通过 SendMessageTool 继续 Worker
          通过 TaskStopTool 终止 Worker
```

### 4.2 子 Agent 生命周期

```
AgentTool.call()
  ├── 分配 agentId（UUID）
  ├── createSubagentContext()    隔离 messages + fileStateCache
  ├── registerAsyncAgent()       → LocalAgentTask（进程内）
  │   或 registerRemoteAgentTask() → RemoteAgentTask（CCR）
  ├── runAgent()
  │   ├── 构建子 Agent SystemPrompt
  │   ├── 装配工具（可配置子集）
  │   └── query()                 独立循环
  └── 完成后通过 task-notification XML 注入主 Agent 消息列表

task-notification 格式（XML，以 user 角色注入）：
  <task-id> <status> <summary> <result> <usage>
```

### 4.3 Worktree 隔离

```
AgentTool（isolation: "worktree"）
  ├── createAgentWorktree()  → git worktree add
  ├── Agent 在独立目录工作
  ├── hasWorktreeChanges()   → 检测修改
  └── removeAgentWorktree()  → 清理（无修改时自动删除）
```

## 五、Skill 系统

```
Skill 来源
├── Bundled      编译进二进制，registerBundledSkill()
├── Disk         ~/.claude/skills/*.md（前置matter描述）
└── MCP          MCP Server 暴露的 prompt/

SkillTool.call(name, args)
  ├── 加载 skill（getPromptForCommand）
  ├── context: 'inline'  → 追加为 user message，当前会话继续
  └── context: 'fork'    → 产生子 Agent（AgentTool）执行
```

Skill 定义结构（`BundledSkillDefinition`）：
- `name / description / aliases / whenToUse`
- `allowedTools` — 限制子 Agent 可用工具集
- `model` — 可指定专用模型
- `files` — 附带参考文件（懒惰解压到磁盘）
- `hooks` — Skill 专属 Hook 脚本

## 七、权限系统

```
PermissionMode
  ├── default     需要用户交互确认（危险操作）
  ├── auto        自动批准大多数操作（--dangerously-skip-permissions-prompts）
  ├── bypass      完全绕过（受控环境）
  └── plan        Plan Mode：只读，禁止写操作

规则来源（按优先级）：
  1. 全局 settings.json（~/.claude/settings.json）
  2. 项目 settings.json（.claude/settings.json）
  3. 本地 settings.local.json
  4. MDM（企业管理策略）

Hook 脚本（pre-tool-use / post-tool-use）：
  Shell 脚本，exit 0 = 允许，exit 2 = 拒绝（含理由）
```

## 九、状态管理

```
AppStateStore（state/AppStateStore.ts）
核心字段：
  ├── messages: Message[]           会话消息历史
  ├── tasks: Map<string, TaskState> 后台任务注册表
```

## 十、关键设计决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 运行时 | Bun | 启动速度快，内置 TS 支持，`feature()` DCE |
| TUI | React + Ink | 声明式 UI，复用 React 生态 |
| 状态管理 | 自制 pub/sub Store | 无 Redux 开销，与 React 18 `useSyncExternalStore` 直接对接 |
| 工具执行 | AsyncGenerator | 支持流式进度输出 |
| 多 Agent | 进程内 + 远程两种模式 | 低延迟（进程内）vs 隔离（远程 CCR） |
| 权限 | 多层规则 + Hook 脚本 | 企业可扩展，用户可自定义 |
| 压缩 | 自动检测 token 阈值 | 长对话不截断，透明续接 |
| Feature 标志 | Bun `feature()` + Statsig + GrowthBook | 构建期 DCE + 运行期灰度 |
