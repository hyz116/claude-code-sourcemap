# Claude Code Skill 系统深度分析

> 分析对象：`cc/src/skills/` + `cc/src/tools/SkillTool/` + `cc/src/commands.ts`

---

Skill 系统核心设计总结：

  1. 三源统一
  Bundled（二进制内置）、Disk（.claude/skills/name/SKILL.md）、MCP（Server prompt）三种来源最终都归一到同一个 Command 数据结构，SkillTool 无需感知来源差异。

  2. 双模执行
  - Inline（默认）：Skill prompt 作为 user message 注入当前会话，LLM 继续响应，上下文共享
  - Fork（context: fork）：通过 runAgent() 产生隔离子 Agent，有独立 token 预算，不污染主会话

  3. allowedTools 自动权限注入
  Skill 声明 allowed-tools: [Bash(git:*)] 后，执行时自动合并到权限上下文 alwaysAllowRules，子 Agent 调用这些工具无需用户确认对话框。

  4. disableModelInvocation 的妙用
  /batch 和 /lorem-ipsum 等 Skill 设置 disableModelInvocation: true——SkillTool 直接将 prompt 内容输出给用户，相当于"展开的指令模板"，LLM 读取后再按指令编排执行。

  5. 条件激活（paths）
  Skill 可通过 paths: [src/components/**] 声明激活条件，每次 FileEditTool 写文件时检查路径匹配，命中则触发 Skill 目录重新扫描并激活——实现"只在相关代码区域才暴露专用工作流"的效果。

  6. /skillify 的元能力
  将当前会话本身捕获为新 Skill 的能力——读取会话记忆 + 用户消息历史，交互式采访用户，生成结构化的 SKILL.md 写入磁盘，让用户的工作流立即可复用。

## 一、Skill 系统全景

Skill 是 Claude Code 的**可重用工作流单元**，本质是一段 Markdown 提示词，通过 `SkillTool`（`/skill-name`）调用后注入当前会话或独立子 Agent 执行。它将"提示词即代码"的理念落地，允许用户和系统将常用操作封装为命名指令。

```
Skill 三大来源
├── Bundled    编译进 CLI 二进制
├── Disk       文件系统目录（project / user / policy）
└── MCP        MCP Server 暴露的 prompt
```

核心类型是 `Command`（type = `'prompt'`），统一承载所有来源的 Skill。

## 二、Command 数据结构

```typescript
// types/command.ts
type PromptCommand = {
  type: 'prompt'
  name: string                        // 调用名（e.g. "simplify"）
  description: string
  aliases?: string[]
  allowedTools: string[]              // 最小权限声明（e.g. ["Bash(git:*)"]）
  argumentHint?: string               // 参数提示（显示在 UI）
  argNames?: string[]                 // 具名参数（$arg_name 替换）
  whenToUse?: string                  // 自动触发描述（SkillTool prompt 中展示）
  model?: string                      // 模型覆盖
  disableModelInvocation: boolean     // true = 直接输出，不走 LLM
  userInvocable: boolean              // false = 隐藏，不出现在用户可选列表
  context?: 'inline' | 'fork'         // 执行上下文
  agent?: string                      // 指定子 Agent 类型
  effort?: EffortValue                // 努力等级（影响 thinking budget）
  paths?: string[]                    // 条件激活路径模式
  hooks?: HooksSettings               // Skill 专属 Hook 脚本
  shell?: FrontmatterShell            // Shell 执行配置
  source: SettingSource               // policySettings | userSettings | projectSettings | plugin
  loadedFrom: LoadedFrom              // skills | commands_DEPRECATED | plugin | managed | bundled | mcp
  skillRoot?: string                  // Skill 目录路径（bundled 解压路径 / disk 目录）
  contentLength: number               // Prompt 大小（用于 token 估算）
  isHidden: boolean                   // 对应 !userInvocable
  getPromptForCommand(                // 核心：返回注入的 prompt 内容
    args: string,
    context: ToolUseContext,
  ): Promise<ContentBlockParam[]>
}
```

## 三、Skill 来源详解

### 3.1 Bundled Skill（内置）

**注册流程**：

```
entrypoints/cli.tsx 启动
  → initBundledSkills()           [skills/bundled/index.ts]
  → 各 register*Skill() 函数
  → registerBundledSkill(def)     [skills/bundledSkills.ts]
  → 追加到内存 bundledSkills[]
```

`BundledSkillDefinition` 与 `Command` 的主要差别：
- `files?: Record<string, string>` — 附带参考文件（延迟解压到临时目录）
- 解压时用 `O_EXCL | O_NOFOLLOW | 0o600` 防止符号链接攻击
- 解压后在 prompt 首行注入 `Base directory for this skill: <dir>`，让模型可 Read/Grep 这些文件

**安全文件解压**（`extractBundledSkillFiles`）：
```
getBundledSkillsRoot()     → 含进程 nonce 的临时目录（防预创建攻击）
mkdir 0o700                → 所有者独占目录
safeWriteFile 0o600        → O_EXCL 防覆盖，O_NOFOLLOW 防末端符号链接
```

### 3.2 Disk Skill（磁盘文件）

**目录搜索优先级**（`getSkillDirCommands(cwd)`）：

```
1. Policy    <managed-path>/.claude/skills/
2. User      ~/.claude/skills/
3. Project   .claude/skills/  （从 cwd 向上遍历到 $HOME）
4. --add-dir 额外目录
```

**文件格式**（唯一支持）：

```
.claude/skills/
  my-workflow/
    SKILL.md      ← 必须叫 SKILL.md，父目录名即 skill 名
```

`SKILL.md` 内容结构：

```markdown
---
name: 可选显示名
description: 一行描述（必填或从 Markdown 提取首句）
allowed-tools:
  - Bash(git:*)
  - Read
argument-hint: "<pr-number>"
arguments:
  - pr_number
when_to_use: |
  Use when user wants to... Examples: 'cherry-pick', 'CP this PR'
model: sonnet
effort: high
context: fork
agent: general-purpose
paths:
  - src/**
  - tests/**
user-invocable: true
hooks:
  PostToolUse:
    - matcher: Write
      hooks:
        - type: command
          command: prettier --write $FILE
shell:
  type: bash
  timeout: 30
---

# Skill Title

## Inputs
- `$pr_number`: PR 编号

## Steps
...
```

**Frontmatter 解析**（`parseSkillFrontmatterFields`）关键字段：

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `name` | string | - | 显示名（不影响调用名） |
| `description` | string | 从 Markdown 首句提取 | SkillTool 展示用 |
| `allowed-tools` | string[] | [] | 最小权限白名单 |
| `argument-hint` | string | - | UI 提示（如 `<branch>`） |
| `arguments` | string[] | [] | 具名参数（`$name` 替换） |
| `when_to_use` | string | - | 触发描述，影响 SkillTool 自动调用 |
| `model` | string | inherit | 'sonnet' / 'opus' / 'haiku' / 完整 ID |
| `effort` | string/int | - | low/medium/high/max/整数 |
| `context` | 'fork' | inline | 'fork' = 独立子 Agent |
| `agent` | string | - | 子 Agent 类型名 |
| `paths` | string[] | - | 条件激活路径（ignore 库格式） |
| `user-invocable` | bool | true | false = 隐藏，只能程序化调用 |
| `disable-model-invocation` | bool | false | 直接输出，不走 LLM |
| `hooks` | HooksSettings | - | Skill 专属钩子 |
| `version` | string | - | Skill 版本号 |

### 3.3 MCP Skill

**创建流程**：

```
MCP Server 连接成功
  → fetchToolsForClient()（MCP prompt 列表）
  → getMCPSkillBuilders().createSkillCommand()
       ↑ 解耦设计：通过 mcpSkillBuilders.ts 写时注册，避免循环依赖
         （client.ts → mcpSkills → loadSkillsDir → … → client.ts）
  → 追加到 AppState.mcp.commands
  → SkillTool 的 getAllCommands() 合并 MCP skills
```

**MCP Skill 安全限制**：
- Prompt 中 `!`command`` / ` ```! ``` ` 语法**不执行**（Shell 注入防护）
- `${CLAUDE_SKILL_DIR}` 不替换（无意义）
- 仅 `loadedFrom !== 'mcp'` 时才执行内联 Shell

### 3.4 Legacy Commands（已废弃）

```
.claude/commands/
  my-cmd.md              → skill 名: my-cmd
  subdir/
    SKILL.md             → skill 名: subdir:my-cmd（父目录名）
    my-cmd.md            → skill 名: subdir:my-cmd

命名规则：namespace = 父目录 相对于 commands/ 的路径，: 分隔
```

`loadedFrom = 'commands_DEPRECATED'`，功能与 `/skills/` 完全兼容，差别仅是支持单文件格式。

## 四、Skill 加载与去重

```
getSkillDirCommands(cwd)    memoize（按 cwd 缓存，会话内不重复扫描）
  │
  ├── loadSkillsFromSkillsDir(managedPath)
  ├── loadSkillsFromSkillsDir(userPath)
  ├── for each projectPath: loadSkillsFromSkillsDir(path)
  ├── loadSkillsFromCommandsDir(cwd)   -- 兼容旧 /commands/
  │
  └── 去重（realpath 解析符号链接 → 比较规范路径）
        原因：覆盖挂载、符号链接、NFS inode=0 等导致同一文件多路径
```

**条件激活**（`paths` 字段）：

```
skill.paths = ["src/**", "tests/**"]

activateConditionalSkillsForPaths(editedPaths)
  → FileEditTool 每次写文件后触发
  → 检查 editedPath 是否命中任一 skill.paths 模式
  → 命中 → addSkillDirectories(dir) 信号 → getSkillDirCommands 缓存失效 → 重新加载
```

## 五、Prompt 内容生成（getPromptForCommand）

```
disk skill getPromptForCommand(args, context):

1. 拼接 Base directory 头部（若有 skillRoot）
   → "Base directory for this skill: /path/to/skill\n\n"

2. 具名参数替换（substituteArguments）
   → $pr_number → 实际参数值
   → 位置参数 $1 $2 ...（旧格式，向后兼容）

3. 变量替换
   → ${CLAUDE_SKILL_DIR} → skill 目录路径（Windows 转正斜杠）
   → ${CLAUDE_SESSION_ID} → 当前会话 ID

4. 内联 Shell 执行（仅 loadedFrom !== 'mcp'）
   → !`command` 或 ```! ... ``` 语法
   → executeShellCommandsInPrompt()
   → 使用 allowedTools 权限上下文
   → 输出替换进 prompt

5. 返回 ContentBlockParam[]（通常单个 text block）
```

## 六、SkillTool 执行路径

### 6.1 SkillTool 接口

```typescript
// SkillTool 输入
{ skill: string, args?: string }

// 输出（联合类型）
| { success, commandName, allowedTools?, model?, status: 'inline' }
| { success, commandName, status: 'forked', agentId, result }
```

### 6.2 路由决策

```
SkillTool.call({ skill, args }, context)
  │
  ├── getAllCommands(context)     本地 + MCP skill
  ├── findCommand(skill)         精确匹配 + 别名
  ├── 权限检查（allowedTools ∩ permissionContext）
  │
  ├── command.context === 'fork'
  │     └── executeForkedSkill()   ← 独立子 Agent
  │
  └── default（inline）
        └── executeInlineSkill()   ← 注入当前会话
```

### 6.3 Inline 执行（默认）

```
executeInlineSkill():
  ├── command.getPromptForCommand(args, context)
  │     → ContentBlockParam[]
  ├── prepareForkedCommandContext()  准备消息格式
  ├── 注入为 user message 追加到 context.messages
  ├── 返回 { status: 'inline', allowedTools, model }
  └── query() 循环继续（Skill 内容作为用户消息处理）

特殊路径（disableModelInvocation = true）：
  → 直接返回 getPromptForCommand 内容给用户
  → 不经过 LLM（例如 /batch 和 /lorem-ipsum 的元指令）
```

### 6.4 Fork 执行（context: fork）

```
executeForkedSkill():
  ├── 分配 agentId
  ├── prepareForkedCommandContext()   隔离消息上下文
  ├── 合并 effort 到 agentDefinition
  ├── runAgent({
  │     agentDefinition,
  │     promptMessages,             ← Skill 内容作为初始消息
  │     toolUseContext: {
  │       ...context,
  │       getAppState: modifiedGetAppState  ← 修改 allowedTools 权限
  │     },
  │     isAsync: false,             ← 同步等待（非后台）
  │     model: command.model,
  │   })
  ├── 流式收集 agentMessages
  ├── onProgress → SkillToolProgress（实时 TUI 进度）
  ├── extractResultText(agentMessages)
  └── 返回 { status: 'forked', agentId, result }
```

### 6.5 allowedTools 注入机制

Fork 执行时，`modifiedGetAppState` 将 `command.allowedTools` 合并进权限上下文：

```
{
  ...appState,
  toolPermissionContext: {
    ...toolPermissionContext,
    alwaysAllowRules: {
      ...alwaysAllowRules,
      command: allowedTools,       ← Skill 声明的工具自动放行
    }
  }
}
```

这使得 skill 能声明最小权限而不触发用户确认对话框。

### /simplify 深度解析

```markdown
简化 Skill 的并行结构：
  Phase 1: git diff 获取变更
  Phase 2: AgentTool 同时启动 3 个 Worker
    ├── Agent 1: 代码复用审查（搜索已有 utility）
    ├── Agent 2: 代码质量审查（反模式检测）
    └── Agent 3: 效率审查（N+1、热路径、内存泄漏）
  Phase 3: 等待全部完成 → 聚合问题 → 直接修复
```

### /batch 深度解析

```markdown
批量变更 Skill（disableModelInvocation = true）：
  → SkillTool 直接输出 prompt，不经过 LLM
  → 用户看到: 完整的编排指令文本
  → LLM 读取该指令后执行：

  Phase 1 (Plan Mode):
    EnterPlanModeTool →
    研究子 Agent →
    分解 5-30 个独立工作单元 →
    确定 e2e 测试方案 →
    ExitPlanModeTool（用户确认计划）

  Phase 2 (Spawn Workers):
    AgentTool × N（all isolation: worktree, run_in_background: true）
    每个 Worker 提示词包含：
      目标指令 + 具体任务 + 约定 + e2e 方案 + 统一工作流
    统一工作流：simplify → 单测 → e2e → commit/push → 创 PR → 报告 PR URL

  Phase 3 (Track Progress):
    渲染状态表 → 收 task-notification → 更新表格 → 汇总
```

### /skillify 深度解析

```markdown
Session → Skill 捕获工具（disableModelInvocation = false）：

  getPromptForCommand():
    ├── getSessionMemoryContent()   读取会话记忆摘要
    ├── extractUserMessages()       提取当前会话所有用户消息
    └── 注入到 SKILLIFY_PROMPT 模板

  执行流程（inline）：
    Round 1: AskUserQuestion 确认 Skill 名和目标
    Round 2: 确认步骤/参数/位置/执行模式（inline/fork）
    Round 3: 逐步细化（成功标准/并发/工具/检查点）
    Round 4: 触发短语 + 注意事项
    → 生成 SKILL.md 内容 → AskUserQuestion 确认 → Write 到磁盘
```

## 八、Skill 发现与上下文感知

### 8.1 SkillTool 的 Prompt（展示给模型的 Skill 列表）

```
SkillTool.prompt = getPrompt(projectRoot)

输出内容：
  所有可用 Skill 的简介：
    - name: skill 调用名
    - description: 一行描述
    - whenToUse: 触发场景（这是关键！让模型知道何时主动调用）
    - argumentHint: 参数提示

  模型根据 whenToUse 判断是否应该主动调用 Skill
```

### 8.2 条件激活（paths 字段）

```
skill:
  name: my-frontend-skill
  paths:
    - src/components/**
    - src/pages/**

当 FileEditTool 写入 src/components/Button.tsx 时：
  activateConditionalSkillsForPaths(["/abs/path/to/src/components/Button.tsx"])
    → discoverSkillDirsForPaths()
    → 找到包含 my-frontend-skill 的 skills/ 目录
    → addSkillDirectories(dir) 信号广播
    → getSkillDirCommands memoize 缓存失效
    → 下次 SkillTool 调用时重新加载，包含该 skill
```

### 8.3 --bare 模式

```
isBareMode() = true:
  ├── 跳过 managed/user/project 目录自动扫描
  ├── 只加载 --add-dir 显式目录
  └── Bundled skills 不受影响（独立注册）
```

## 九、Skill 版本与去重优先级

```
同名 Skill 的优先级（低 → 高，高优先级覆盖低优先级）：

  managed (policy)  > user  > project  > --add-dir

去重机制：
  realpath() 解析符号链接 → 比较规范路径
  相同规范路径的文件只加载一次（防止 --add-dir 与 project 重复加载同一文件）

Legacy /commands/ 兼容：
  同一目录同时有 skills/ 和 commands/ → 合并加载
  同名 skill：loadedFrom = 'skills' 优先于 'commands_DEPRECATED'
```

## 十、设计模式总结

| 模式 | 实现 | 目的 |
|------|------|------|
| **提示词即代码** | SKILL.md 的 Markdown 格式 | 非工程师也能编写和维护 Skill |
| **最小权限声明** | `allowed-tools` 自动注入权限上下文 | Skill 运行时无需用户手动确认 |
| **双模执行** | inline / fork 二选一 | 前者共享上下文，后者隔离预算 |
| **动态参数替换** | `$name` 具名 + `$1` 位置 | 将同一流程参数化为通用 Skill |
| **Shell 预处理** | `!`cmd`` 内联 Shell | 动态内容（git log、env var）注入 prompt |
| **条件激活** | `paths` + FileEdit 事件 | 只在相关文件工作时暴露特定 Skill |
| **懒加载去重** | memoize + realpath | 避免会话内重复扫描磁盘 |
| **MCP 解耦** | mcpSkillBuilders 写时注册 | 打破 client.ts ↔ loadSkillsDir 循环依赖 |
| **disableModelInvocation** | batch/lorem-ipsum 使用 | 让 Skill 本身就是输出，不再嵌套 LLM |
| **安全文件解压** | O_EXCL + O_NOFOLLOW + nonce 目录 | Bundled Skill 文件提取防竞争攻击 |
